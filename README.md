# Reaction Standee

ショート動画向けの「立ち絵リアクション演出アプリ」です。Webカメラで大きなポーズを認識し、キャラクターの立ち絵差分をリアクションとして切り替えます。

口パク、音声入力、ホットキー操作はv1には入れていません。ユーザーの体を完全追従するのではなく、ポーズをキャラクター演出の入力コマンドとして扱います。

## 起動

```bash
npm install
npm run dev
```

設定画面:

```text
http://127.0.0.1:5173/settings
```

OBS・録画用の表示画面:

```text
http://127.0.0.1:5173/avatar
```

Safari/Chromeのウィンドウキャプチャ用画面:

```text
http://127.0.0.1:5173/capture
```

初回トラッキング時に、MediaPipe Pose LandmarkerのWASMとモデルをブラウザが読み込みます。

## v1機能

- MediaPipe Pose LandmarkerによるWebカメラポーズ認識
- 5種類のリアクション画像登録
- 通常時のまばたき、呼吸、ゆらぎによる生命感演出
- 登録画像の個別削除と全削除
- 画像切替時のCSSアニメーション
- キャラ画像のサイズ・位置調整
- 背景設定: 透明、グリーンバック、任意色
- 背景画像の登録と合成表示
- 録画比率切り替え: 9:16、16:9
- 透明PNG向けの白フチ表示
- カメラ選択
- トラッキングON/OFF
- 判定感度: 低、標準、高
- `/settings` と `/avatar` の2画面構成
- `/capture` によるUIなし・顔プレビューなしのトラッキング表示
- ローカルAPIによるリアクション状態とOBS表示設定の共有
- localStorageによる設定保存
- IndexedDBによる画像保存

## 画像差分

設定画面から以下5枚を登録します。

```text
normal.png
joy.png
surprised.png
troubled.png
explain.png
```

推奨は背景透過PNGです。背景抜きは外部の画像編集アプリで行い、このアプリには透過済みPNGを登録してください。OBSでクロマキーを使う場合は、アプリ側の背景をグリーンバックに変更できます。

OBSのブラウザソースはChrome/Safariとは別のブラウザ保存領域を使うため、設定画面でアップロードした画像がOBS側に共有されない場合があります。その場合は、同じファイル名で `public/reactions/` にPNGを配置してください。アプリはブラウザ内のアップロード画像が無い時に、`/reactions/normal.png` などの静的ファイルを自動で読みます。

## 背景画像と録画比率

設定画面の表示セクションで、録画比率を `9:16` または `16:9` に切り替えられます。初期値はショート動画向けの `9:16` です。

背景は以下から選べます。

- 透明
- グリーンバック
- 任意色
- 背景画像

背景画像は設定画面から登録できます。登録した背景画像はIndexedDBに保存され、キャラクターの背面で録画フレーム全体を埋めるようにcover表示されます。OBS録画で背景込みの完成画面を作りたい場合は、背景を `背景画像` にしてください。

## 生命感演出

設定画面の生命感セクションで、通常状態だけに生命感エフェクトを設定できます。

- 生命感エフェクトON/OFF
- 通常まばたき画像の登録
- 目元範囲のX/Y/幅/高さ調整
- まばたきON/OFF
- 呼吸・ゆらぎON/OFF
- エフェクトの強さ: 弱、標準、強

まばたきは通常表示中だけ、`normal` の目閉じ差分画像から目元範囲だけを切り抜き、通常画像の上に短時間だけ重ねます。通常画像そのものは固定されるため、AI生成差分で口元、髪、服などが微妙に変わる場合でも、全身差し替えよりチラつきを抑えられます。

目閉じ差分が未登録の場合は何もせず、エラーにはなりません。喜び、驚き、困惑、説明などのリアクション画像には、目閉じ差分登録を求めません。

目閉じ差分は、通常画像と同じ構図、同じサイズの全身PNGを想定しています。設定画面の黄色い枠を目元に合わせると、その範囲だけがまばたき時に表示されます。

## 画面の役割

- `/settings`: 画像登録、カメラ選択、表示調整、デバッグ確認を行う設定画面です。顔プレビューと設定UIが表示されます。
- `/capture`: Safari/Chromeのウィンドウキャプチャ向け画面です。設定UIと顔プレビューを表示せず、キャラだけを表示しながら同じページ内でポーズトラッキングします。
- `/avatar`: OBS Browser Source向けの表示専用画面です。カメラは起動せず、`/settings` または `/capture` から共有されたリアクション状態を表示します。

ショート動画制作や配信では、現状 `/capture` をSafari/Chromeで開き、OBSのウィンドウキャプチャで取り込む方法を推奨します。OBS Browser Sourceは環境によってCSSアニメーションや画像切替が重くなることがあります。

## 認識するポーズ

- `normal`: 手が下がっている、または特別なポーズをしていない
- `joy`: 両手を大きく上げる、ばんざいポーズ
- `surprised`: 片手または両手が顔の近くにある
- `troubled`: 片手が頭の近くにある。頭の傾きも補助的に見ます
- `explain`: 人差し指を立てる

## 安定化

ポーズ候補が短時間だけ揺れた場合は、すぐには切り替えません。

- confidence thresholdで弱い判定を保留
- debounceで同じ候補が続いた時だけ切り替え
- cooldownで連続切り替えを抑制
- 通常状態へ戻る時は少し長めに待機
- カメラから一瞬外れても即座にはnormalに戻さない

## OBSで録画する 推奨

1. `npm run dev` で起動します。
2. Chrome/Safariで `/settings` を開き、画像、カメラ、背景、サイズ、位置を設定します。
3. 背景込みで録画する場合は背景画像を登録して、背景を `背景画像` にします。クロマキー運用の場合はグリーンバックにします。
4. Safari/Chromeで `/capture` を開きます。
5. `/capture` でカメラを許可し、ポーズ切替が動くことを確認します。
6. OBSでソースを追加し、`macOS スクリーンキャプチャ` または `ウィンドウキャプチャ` を選びます。
7. `/capture` を開いているSafari/Chromeウィンドウを選択します。
8. OBS上で必要に応じて上部のブラウザUIをクロップします。macOSではソース選択後、Optionキーを押しながら赤枠の端をドラッグすると切り抜けます。
9. OBSのキャンバスと出力解像度をショート動画向けに `1080x1920` にします。
10. 必要ならOBS側でクロマキーを追加してグリーンバックを抜きます。
11. `録画開始` で録画し、`ファイル > 録画を表示` から録画ファイルを確認します。

OBSのBrowser Sourceではなく、Safari/Chromeのウィンドウをキャプチャする理由は、OBS内ブラウザでCSSアニメーションや画像切替が重くなる環境があるためです。Safari/Chromeで直接動かすと滑らかな場合は、この方式が安定します。

## OBS Browser Sourceで使う 代替

OBS Browser Sourceを使う場合は、OBSのブラウザソースに `/avatar` を指定します。

```text
http://127.0.0.1:5173/avatar
```

この場合、カメラトラッキングは `/settings` または `/capture` 側で行い、`/avatar` はローカルAPI `/api/reaction` 経由で共有されたリアクション状態と表示設定を受け取ります。

注意点:

- OBS Browser SourceはSafari/Chromeとは別の保存領域を使います。
- アップロード画像が共有されない場合があります。その場合は `public/reactions/` に `normal.png` などを配置してください。
- 環境によってCSSアニメーションや画像切替が遅くなることがあります。

## Safariのカメラ許可とURL

開発中はURLを `127.0.0.1` に統一することを推奨します。

```text
http://127.0.0.1:5173/capture
```

Safariでは `127.0.0.1` と `localhost` が別サイト扱いになるため、画像保存、設定、カメラ許可も別々になります。`localhost` で開くと、`127.0.0.1` で登録した画像が表示されないことがあります。

カメラ許可はSafariの `設定 > Webサイト > カメラ` で `127.0.0.1` を許可できます。ただしローカルサイトでは環境によって毎回確認されることがあります。

## トラブルシューティング

- 画像が表示されない: `127.0.0.1` と `localhost` を混在させていないか確認してください。必要なら `/settings` で画像を再登録します。
- `/api/reaction` が画面表示になる: `vite.config.js` など古い生成ファイルが残っていないか確認してください。現在は `vite.config.ts` を使います。
- OBS Browser Sourceが遅い: `/capture` をSafari/Chromeで開き、OBSのウィンドウキャプチャで取り込んでください。
- 画面が真っ白になる: ページを再読み込みしてください。エラー画面が出た場合は表示されたメッセージを確認します。
