# デモキャラクター画像生成依頼文

以下をChatGPTの画像生成へ、そのまま依頼するためのマスタープロンプトです。

```text
「Reaction Standee」というWebアプリの公開デモ用に、オリジナルキャラクターの立ち絵一式を制作してください。

目的:
Webカメラで認識したポーズに応じて、通常、喜び、驚き、困惑、説明の立ち絵を切り替えます。通常状態では、まばたき、目線、口パク差分を目元・口元だけ重ねて使います。

キャラクター:
- 活動名は「IT勇者」
- 完全オリジナルの、親しみやすいVTuber風男性キャラクター
- 3.5〜4頭身程度の、かわいさと見やすさを両立したちびキャラ
- 若い成人
- ダークブラウンの短めの髪
- 青〜シアン系の瞳
- 細い丸眼鏡
- シルバーの軽装鎧
- 鮮やかな青い短いマント
- ベルト付近に小型の魔法端末
- 鎧や装備に、ごく控えめなシアンの回路・魔法陣モチーフ
- 明るく知的で、ITを分かりやすく案内してくれそうな印象
- 既存作品、既存VTuber、企業ロゴ、実在製品のデザインには似せない

共通仕様:
- すべて正方形2048×2048px
- 背景は完全透過PNG
- 白フチ、影、床、背景、小物の背景演出、文字、ロゴ、透かしは入れない
- 頭や足、マント、手を切らず、全身をキャンバス内に収める
- すべて同じカメラ角度、同じ頭身、同じキャラクターデザイン、同じ色、同じ塗り、同じ光源
- キャラクターの大きさと足元位置を全画像で揃える
- 画面下端には透明の余白を十分に残す
- 後からアプリ側で白フチを付けるため、画像自体にはフチを付けない
- 最終画像は1枚の一覧表やスプライトシートにまとめず、ファイルごとに独立したPNGとして出力する

まず最初に、正面〜やや斜め向きの全身キャラクターデザイン案を1枚作り、衣装、顔、色、頭身を確定してください。
確定後、その画像を基準参照として以下を順番に制作してください。

リアクション画像:
1. normal.png
   自然体。穏やかな微笑み。腕は体の横または軽く前に置く。

2. joy.png
   嬉しそうな笑顔。両手を上げた、明るく大きな喜びのポーズ。

3. surprised.png
   目を少し大きく開き、片手または両手を顔の横へ近づけた驚きのポーズ。

4. troubled.png
   少し困った表情。片手を頭の近くへ置き、考え込むようなポーズ。

5. explain.png
   自信のある優しい表情。片手の人差し指を立てて説明するポーズ。

通常画像の編集差分:
以下はnormal.pngを新しく描き直さず、normal.pngそのものを画像編集してください。指定部位以外の髪、輪郭、眼鏡、体、服、手、マント、色、影、透明範囲、座標は1ピクセルも変えないでください。

6. normal_blink.png
   normal.pngから、両目だけを自然に閉じる。

7. eye_look_left.png
   normal.pngから、瞳・黒目の向きだけをキャラクター視点の左へ動かす。

8. eye_look_right.png
   normal.pngから、瞳・黒目の向きだけをキャラクター視点の右へ動かす。

9. mouth_small_open.png
   normal.pngから、口だけをごく自然な小開き口へ変更する。

10. mouth_wide_open.png
    normal.pngから、口だけを自然な発話中の大開き口へ変更する。叫び顔にはしない。

進め方:
- 一度に一枚の完成画像を確認しながら進める
- 毎回、直前の基準画像を参照して同一人物・同一衣装を維持する
- 通常差分5枚は必ず画像編集で作り、全身を再生成しない
- 各画像を出力する前に、透明背景、2048×2048、全身が切れていないこと、下端の透明余白、ファイル名を確認する
- 途中でデザインを勝手に変更しない

まずはキャラクターデザイン案とnormal.pngの候補から始めてください。
```

## 生成後の確認

- 10枚すべてが同じ解像度か。
- PNGの背景が本当に透過しているか。
- 画像下端に透明余白があるか。
- 通常差分で、目や口以外が変化していないか。
- ファイル名がアプリの登録先と対応しているか。

画像生成AIは、複数画像を一度に頼むと細部が変化しやすいため、同じチャット内でマスタープロンプトを渡し、完成した基準画像を参照しながら一枚ずつ進める方法を推奨します。
