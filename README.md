# Reaction Standee

Webカメラで大きなポーズを認識し、2Dキャラクターの立ち絵差分を切り替えるリアクション演出アプリです。

体を完全追従するのではなく、ポーズを「喜び」「驚き」「困惑」「説明」などを呼び出す入力コマンドとして使います。通常状態には、まばたき、呼吸、目線、音声連動の口パクを加えられます。

ショート動画の録画、解説動画、配信画面などで、紙芝居より少し生きているように見える立ち絵表現を目指しています。

## デモ

GitHub Pagesで公開しているWeb版です。

- [Reaction Standeeを開く](https://it-yusha.github.io/reaction-standee/)
- [録画表示を直接開く](https://it-yusha.github.io/reaction-standee/?route=record)

動作GIFまたは短いデモ動画は、デモキャラクター素材の準備後に追加します。

## 使い方

1. 設定画面で通常、喜び、驚き、困惑、説明の透過PNGを登録します。
2. トラッキングをONにして、ブラウザのカメラ使用を許可します。
3. 表示サイズ、位置、背景、白フチ、生命感、口パクを調整します。
4. 「録画表示」を開き、ポーズでリアクションが切り替わることを確認します。

公開版の設定と画像はブラウザ内に保存されます。別ブラウザやDock版へ移す場合は、画像セクションの「設定一式を書き出す」と「設定一式を読み込む」を使います。

## 主な特徴

- 体の完全追従ではなく、ポーズをリアクション呼び出しコマンドとして扱う設計
- 音量を直接口の開きにせず、発話判定とリズム制御で自然に見える口パクを実装
- まばたき、目線、口パクを全身差し替えではなく、目元/口元範囲のオーバーレイとして実装
- 通常状態の呼吸、待機ランダム、発話中ゆらぎ、カメラ追従で軽い生命感を追加
- Safari、Chrome、OBS Browser Source、Electron、WKWebViewを比較し、DOM/CSS品質を維持しやすい録画導線を検証

## プライバシー

カメラ映像とマイク音声は、ポーズ認識と口パク判定のためにブラウザ内で処理します。このアプリ自体は、カメラ映像や音声をサーバーへ送信・保存しません。

GitHub Pages公開版で登録した画像や設定は、基本的にアクセスしたブラウザ内に保存されます。ローカル開発版の共有保存 `.reaction-standee/` は、自分のMacでの制作・録画向けの補助機能です。

### PWAとして使う

公開版にはWeb App ManifestとService Workerを含めています。Safariでは共有メニューの「Dockに追加」、iPhone/iPadでは「ホーム画面に追加」を使うと、ブラウザの通常タブよりアプリに近い形で起動できます。

PWA化しても、カメラ映像やマイク音声を外部サーバーへ送る構成にはなりません。登録画像と設定は、追加したWebアプリを含む各ブラウザ環境内に保存されます。

SafariからDockへ追加したWebアプリは、Safari本体とは別の保存領域を使います。Safariで設定済みの場合は、Safari側で「設定一式を書き出す」を実行し、Dock版の設定画面で読み込んでください。この初期設定が必要なため、Dock版は録画画面ではなく設定画面から起動します。設定後に「録画表示」を押すと同じウィンドウで録画表示へ移動し、`Esc` キーで設定画面へ戻れます。

初回表示にはネットワーク接続が必要です。一度読み込んだアプリ本体はキャッシュされますが、MediaPipeのモデルなど外部配信リソースを使うため、完全なオフライン動作は現時点では保証していません。

### ローカル版との関係

ローカル版とGitHub Pages版は、別々のアプリではありません。React、カメラトラッキング、MediaPipe、キャラクター表示、ポーズ切替、生命感、口パク、設定画面は同じソースコードを使います。

- ローカル版: ViteのローカルAPIを使い、画像・設定・リアクション状態をSafari、WKWebView、OBSなどで共有します。
- GitHub Pages版: 静的ホスティングのためローカルAPI通信を行わず、IndexedDBとlocalStorageを使います。
- PWA: GitHub Pages版を独立したウィンドウやホーム画面から起動しやすくする公開方法です。

環境差は保存・同期層とPWA登録に限定し、表示機能を二重実装しない方針です。設計判断の経緯は [docs/decision-log.md](docs/decision-log.md) に記録しています。

## 画像差分

設定画面から以下5枚を登録します。

公開デモ用のオリジナル素材を画像生成で用意する場合は、[デモキャラクター画像生成依頼文](docs/demo-character-generation-prompt.md) を利用できます。

```text
normal.png
joy.png
surprised.png
troubled.png
explain.png
```

推奨は背景透過PNGです。背景抜きは外部の画像編集アプリで行い、このアプリには透過済みPNGを登録してください。OBSでクロマキーを使う場合は、アプリ側の背景をグリーンバックに変更できます。

画像は設定画面で登録すると、ブラウザ内保存に加えて `.reaction-standee/assets/` に共有保存されます。これによりSafari、Chrome、Electron版の表示画面で同じ画像を使いやすくなります。アプリはアップロード画像が無い時に、従来通り `/reactions/normal.png` などの静的ファイルも自動で読みます。

## 背景画像と録画比率

設定画面の表示セクションで、録画比率を `9:16` または `16:9` に切り替えられます。初期値はショート動画向けの `9:16` です。

背景は以下から選べます。

- 透明
- グリーンバック
- 任意色
- 背景画像

背景画像は設定画面から登録できます。登録した背景画像は共有ローカル保存され、キャラクターの背面で録画フレーム全体を埋めるようにcover表示されます。OBS録画で背景込みの完成画面を作りたい場合は、背景を `背景画像` にしてください。

## 生命感演出

設定画面の生命感セクションで、通常状態だけに生命感エフェクトを設定できます。

- 動きの強さ: OFF、弱、標準、強、確認用
- 基本: まばたき、呼吸・ゆらぎ、口パク連動ゆらぎON/OFF
- 反応: 待機ランダム、目線差分、カメラ追従ON/OFF
- 調整表示: 位置調整枠ON/OFF
- 差分画像: 通常まばたき、目線左、目線右の登録
- 範囲調整: 目元範囲のX/Y/幅/高さ調整

まばたきは通常表示中だけ、`normal` の目閉じ差分画像から目元範囲だけを切り抜き、通常画像の上に短時間だけ重ねます。通常画像そのものは固定されるため、AI生成差分で口元、髪、服などが微妙に変わる場合でも、全身差し替えよりチラつきを抑えられます。

動きの強さの「確認用」は調整時に動きやカメラ追従を見やすくするための強め設定です。通常の配信・録画では「標準」または「弱」を推奨します。目線差分はまばたきと同じ目元範囲だけを切り抜いて重ねるため、全身画像を切り替えません。

目閉じ差分が未登録の場合は何もせず、エラーにはなりません。喜び、驚き、困惑、説明などのリアクション画像には、目閉じ差分登録を求めません。

目閉じ差分は、通常画像と同じ構図、同じサイズの全身PNGを想定しています。設定画面の黄色い枠を目元に合わせると、その範囲だけがまばたき時に表示されます。黄色い目元枠とオレンジの口元枠は、設定画面の「位置調整枠」で表示/非表示を切り替えられます。

## 口パク

設定画面の口パクセクションで、通常状態だけに音声連動の口パクを設定できます。

- 口パクON/OFF
- マイク音声で動かすON/OFF
- マイク選択
- 発話しきい値
- 小開き口、大開き口の差分画像登録
- 口元範囲のX/Y/幅/高さ調整
- 位置調整枠ON/OFF

通常口は `normal.png` をそのまま使います。マイク音量は口の開き量に直接使わず、発話中かどうかと発話の強さの判定に使います。発話中は通常口、小開き口、大開き口を短いリズムで切り替え、登録した口パク差分画像から口元範囲だけを切り抜いて通常画像の上に重ねます。口パク用差分画像を全身ごと切り替えないため、AI生成差分で髪、服、影などが微妙に変わる場合でもチラつきを抑えられます。

口パク差分は、通常画像と同じ構図、同じサイズの全身PNGを想定しています。設定画面のオレンジ枠を口元に合わせてください。

口パク画像が未登録の場合はエラーにはならず、登録済みの近い口形だけを使います。喜び、驚き、困惑、説明などのリアクション画像には、口パク差分登録を求めません。

## 画面の役割

- `/settings`: 画像登録、カメラ選択、表示調整、デバッグ確認を行う設定画面です。顔プレビューと設定UIが表示されます。通常はここで設定します。
- `/record`: DOM/CSSの高品質表示をそのまま録るための録画専用画面です。設定UIと顔プレビューは表示しません。
- `/canvas`: Canvas出力の実験画面です。Canvasで再描画した映像をWebMとして書き出せますが、DOM/CSS表示と完全一致させる用途ではなく、将来検証用です。
- `/capture`: 旧来のSafari/Chromeウィンドウキャプチャ向け画面です。現在の主導線は `/record` のため、設定画面の上部リンクからは外しています。
- `/avatar`: OBS Browser Source向けの表示専用画面です。OBS内ブラウザの重さが課題になったため、現在の主導線ではありません。必要になった時の代替として残しています。

ショート動画制作や配信では、現状 `/settings` で設定し、WKWebView録画ウィンドウで `/record` をMac側で録画する方法を優先します。Safariの通常ウィンドウ録画も代替として使えます。OBS Browser SourceやChromium系の表示は、環境によってCSSアニメーションや画像切替が重くなることがあります。

判断の経緯は [docs/decision-log.md](docs/decision-log.md) にまとめています。

## WKWebView録画ウィンドウ 推奨

ブラウザのツールバーを避けてDOM/CSS版の `/record` を表示する、Mac録画向けのWKWebViewウィンドウを起動できます。

```bash
npm run wk:record
```

このウィンドウは、ローカルで起動中の `http://127.0.0.1:5173/record?camera=low&inferFps=30` をWKWebViewで開きます。録画されるキャラクター画像の品質は落とさず、ポーズ認識用カメラ入力だけを軽くします。

重要: 現在の `ReactionStandeeWKPreview.app` はWebアプリ本体や開発サーバを内蔵していません。FinderやSpotlightから `.app` を起動する場合でも、先に別途開発サーバを起動しておく必要があります。

```bash
npm run dev -- --host 127.0.0.1
```

GUIから起動したい場合は、初回または更新後に以下を実行します。

```bash
npm run wk:install
```

`~/Applications/ReactionStandeeWKPreview.app` に配置されるため、以降はFinderやSpotlightから開けます。ただしWebアプリ本体は別途 `npm run dev -- --host 127.0.0.1` で起動しておく必要があります。将来的にGUI起動を整える場合は、まずサーバ未起動時の案内を分かりやすくし、必要になれば `.app` からローカルサーバも起動する方式を検討します。

動作確認用にパフォーマンス表示を出したい場合は、URLを引数で渡せます。

```bash
npm run wk:record -- "http://127.0.0.1:5173/record?perf=1"
```

WKWebViewの実用設定を計測する場合は以下を使います。

```bash
npm run wk:record:perf
```

`?perf=1` では、左上に実行環境、描画fps、平均フレーム時間、MediaPipe推論fps、推論時間、カメラ解像度が表示されます。Safari、Chrome、WKWebViewで同じ `/record?perf=1` を開き、体感と数値を比べます。

画像は `.reaction-standee/assets/`、表示設定は `.reaction-standee/settings.json` を読みます。初回移行はSafari/Chromeで `/settings` を開くと自動で行われます。

設定変更は `/settings` から行います。録画ウィンドウは共有設定ファイルを短い間隔で確認するため、背景、位置、サイズ、白フチ、生命感などの表示設定は通常1秒以内に反映されます。画像差し替えも共有保存に書き込まれますが、表示が追いつかない場合は `Cmd+R` で再読み込み、または `Cmd+L` で録画URLを開き直してください。設定画面を開きたい時は `Cmd+,` でSafariの `/settings` を開けます。

WKWebView版は、既存WebアプリをSwiftへ移植するものではなく、Macアプリ側で録画機能も持ちません。macOSの画面録画やOBSのウィンドウキャプチャで、この録画用ウィンドウを収録します。

口パク用マイクは録画側のマイク収録と競合しないよう、エコーキャンセル、ノイズ抑制、自動ゲイン制御を切った音量解析用の入力として扱います。macOSの画面録画で自分の声も入れる場合は、画面録画側のマイク選択も有効にしてください。

macOSの画面録画では、アプリ側でカーソルを隠しても録画結果にカーソルが入ることがあります。録画前にマウスカーソルをWKWebViewウィンドウ外へ移動してから収録してください。

## Canvas出力 実験

`/canvas` を開くと、画面下に録画パネルが表示されます。

1. `/settings` で画像、背景、キャラ位置、口パクなどを設定します。
2. `/canvas` を開き、カメラと必要ならマイクを許可します。
3. `録画開始` を押します。
4. ポーズ切り替えや口パクを行います。
5. `停止` を押し、`ダウンロード` からWebMファイルを保存します。

録画ファイルはCanvasで再描画した背景、キャラ画像、白フチ、目元差分、口元差分、リアクション装飾エフェクトを保存します。ブラウザのツールバーやOBSのクロップ枠は入りません。ただし、Canvas版はDOM/CSS版の劣化コピーとして完成させるのではなく、将来Canvas専用に作り込むかを判断するための実験扱いです。

## 認識するポーズ

- `normal`: 手が下がっている、または特別なポーズをしていない
- `joy`: 両手を大きく上げる、ばんざいポーズ
- `surprised`: 片手または両手が顔の近くにある
- `troubled`: 片手が頭の近くにある。頭の傾きも補助的に見ます
- `explain`: 人差し指を立てる

## 安定化

ポーズ候補が短時間だけ揺れた場合は、すぐには切り替えません。

- confidence thresholdで弱い判定を保留
- debounceで同じ候補が続いた時だけ切り替え
- cooldownで連続切り替えを抑制
- 通常状態へ戻る時は少し長めに待機
- カメラから一瞬外れても即座にはnormalに戻さない

## OBSで録画する 推奨

1. `npm run dev` で起動します。
2. Chrome/Safariで `/settings` を開き、画像、カメラ、背景、サイズ、位置を設定します。
3. 背景込みで録画する場合は背景画像を登録して、背景を `背景画像` にします。クロマキー運用の場合はグリーンバックにします。
4. Safari/Chromeで確認する場合は `/record`、Macの録画用ウィンドウで録る場合は `npm run wk:record` でWKWebView版を開きます。
5. 開いた画面でカメラを許可し、ポーズ切替が動くことを確認します。
6. OBSでソースを追加し、`macOS スクリーンキャプチャ` または `ウィンドウキャプチャ` を選びます。
7. `/record` を開いているSafari/Chromeウィンドウ、またはWKWebView録画ウィンドウを選択します。
8. OBS上で必要に応じて上部のブラウザUIをクロップします。macOSではソース選択後、Optionキーを押しながら赤枠の端をドラッグすると切り抜けます。
9. OBSのキャンバスと出力解像度をショート動画向けに `1080x1920` にします。
10. 必要ならOBS側でクロマキーを追加してグリーンバックを抜きます。
11. `録画開始` で録画し、`ファイル > 録画を表示` から録画ファイルを確認します。

OBSのBrowser Sourceではなく、Safari/Chromeのウィンドウをキャプチャする理由は、OBS内ブラウザでCSSアニメーションや画像切替が重くなる環境があるためです。Safari/Chromeで直接動かすと滑らかな場合は、この方式が安定します。

## OBS Browser Sourceで使う 代替

OBS Browser Sourceを使う場合は、OBSのブラウザソースに `/avatar` を指定します。

```text
http://127.0.0.1:5173/avatar
```

この場合、カメラトラッキングと音声入力は `/settings` または `/capture` 側で行い、`/avatar` はローカルAPI `/api/reaction` 経由で共有されたリアクション状態、口形、表示設定を受け取ります。

注意点:

- OBS Browser SourceはSafari/Chromeとは別のブラウザ保存領域を使いますが、共有ローカル保存が使える環境では登録画像自体は `/api/assets` 経由で読み込めます。
- 環境によってCSSアニメーションや画像切替が遅くなることがあります。

## Safariのカメラ許可とURL

開発中はURLを `127.0.0.1` に統一することを推奨します。

```text
http://127.0.0.1:5173/capture
```

Safariでは `127.0.0.1` と `localhost` が別サイト扱いになるため、画像保存、設定、カメラ許可も別々になります。`localhost` で開くと、`127.0.0.1` で登録した画像が表示されないことがあります。

カメラ許可はSafariの `設定 > Webサイト > カメラ` で `127.0.0.1` を許可できます。ただしローカルサイトでは環境によって毎回確認されることがあります。

## トラブルシューティング

- 画像が表示されない: `127.0.0.1` と `localhost` を混在させていないか確認してください。必要なら `/settings` を一度開き、共有ローカル保存への初回移行を走らせます。
- `/api/reaction` が画面表示になる: `vite.config.js` など古い生成ファイルが残っていないか確認してください。現在は `vite.config.ts` を使います。
- OBS Browser Sourceが遅い: Electron版またはSafari/Chromeの `/record` をウィンドウキャプチャで取り込んでください。
- 画面が真っ白になる: ページを再読み込みしてください。エラー画面が出た場合は表示されたメッセージを確認します。
